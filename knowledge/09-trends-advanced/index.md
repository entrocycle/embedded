## 为什么放在进阶专题

这些主题不一定是每个初学者的第一优先级，但在真实产品中非常常见：

- 数据要持久化，需要 Flash、EEPROM、文件系统和掉电保护。
- 设备要有人机交互，需要屏幕、触摸、GUI 框架和渲染性能。
- 设备要本地智能，需要模型量化、部署和传感器数据链路。
- 设备要长期上线，需要安全启动、签名、密钥、可靠性和故障恢复。

## 学习顺序

建议在完成以下基础后再进入本章：

1. C 语言、编译链接和内存模型。
2. MCU 裸机外设和 DMA。
3. RTOS 或 Linux 的基础工程。
4. 网络、OTA 和调试方法。

## 进阶能力验收

| 能力 | 验收问题 |
|---|---|
| 存储 | 断电时配置写到一半怎么办？ |
| 文件系统 | LittleFS、FAT、ext4、squashfs 该怎么选？ |
| GUI | 屏幕刷新卡顿时怎么判断是 SPI、DMA、渲染还是内存问题？ |
| TinyML | 模型放不进 RAM 时如何裁剪、量化和测量？ |
| 安全 | OTA 包如何防篡改、防回滚、防断电损坏？ |
| 可靠性 | 现场偶发死机如何留下足够证据？ |

## 项目建议

- MCU + LittleFS 配置系统。
- LVGL 仪表盘。
- Qt HMI Linux 屏幕应用。
- 加速度计活动识别 TinyML demo。
- 安全 Bootloader + 签名 OTA。
- 现场诊断和可靠性测试套件。

## 深挖问题

- 文件系统选型时，掉电安全、磨损均衡、目录能力和 RAM 占用如何权衡？
- GUI 卡顿时，瓶颈来自总线带宽、DMA、渲染算法、字体资源还是任务调度？
- TinyML 的准确率、延迟、RAM、Flash、功耗之间如何取舍？
- 安全启动、密钥管理、调试口关闭和 OTA 回滚之间有哪些冲突？

## 阶段产出

- 一个存储方案对比表，说明介质、文件系统、掉电保护和寿命估算。
- 一个 GUI 或 TinyML demo 的资源测量记录，包含 RAM、Flash、帧率或推理耗时。
- 一份安全可靠性检查清单，覆盖启动、升级、密钥、调试口、故障恢复和现场诊断。
