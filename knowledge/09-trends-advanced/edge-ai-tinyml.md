# Edge AI 与 TinyML

Edge AI 的目标是在设备侧完成推理，减少延迟、带宽和隐私风险。TinyML 是 Edge AI 在 MCU 等资源受限设备上的形态，核心挑战是模型、内存、算力、功耗和传感器数据链路的平衡。

## 本章任务与边界

完成本章后，你应能定义输入/模型/输出/更新契约，测量 Flash、tensor arena、推理尾延迟和能量，比较误报与漏报代价，并用现场数据漂移、传感器替换和模型 OTA 验证系统。

本章聚焦部署工程，不教授完整机器学习理论。离线准确率不能直接外推到现场，模型输出也不应在缺乏独立保护时直接驱动危险执行器。

完成证据：数据版本与采集条件、训练/现场分布对比、资源和延迟测量、误报漏报矩阵、模型/预处理兼容与回滚实验。

## 适合场景

- 关键词唤醒。
- 手势识别。
- 振动异常检测。
- 简单图像分类。
- 人体活动识别。
- 传感器异常检测。
- 预测性维护。

不适合把复杂大模型直接塞进 MCU。要从数据、模型和产品目标出发做取舍。

## 开发链路

```text
采集数据
  -> 标注
  -> 训练
  -> 量化
  -> 转换
  -> 部署
  -> MCU/边缘推理
  -> 性能和准确率验证
```

工程上最容易低估的是数据采集和标注质量。模型部署不是第一步。

## 模型部署指标

| 指标 | 说明 |
|---|---|
| Flash 占用 | 模型权重和推理代码 |
| RAM 占用 | tensor arena、中间激活、输入输出 buffer |
| 推理时间 | 单次 inference 延迟 |
| 平均功耗 | 推理周期和休眠策略 |
| 准确率 | 离线和现场数据差异 |
| 鲁棒性 | 噪声、温度、个体差异 |

## TensorFlow Lite Micro 流程

典型步骤：

1. 训练 TensorFlow/Keras 模型。
2. 转换为 TFLite。
3. 量化为 int8。
4. 用 `xxd -i` 或构建脚本转成 C 数组。
5. 配置 op resolver。
6. 分配 tensor arena。
7. 填充输入，调用 `Invoke()`。
8. 解析输出。

注意：

- 只注册需要的算子，减少代码大小。
- tensor arena 大小要通过实际运行测量。
- int8 量化需要代表性数据集。
- 输入预处理要和训练保持一致。

## 数据链路

以振动检测为例：

```text
加速度计
  -> 定时采样
  -> 环形缓冲
  -> 窗口切片
  -> 去均值/归一化
  -> FFT 或特征提取
  -> 模型推理
  -> 结果平滑
  -> 告警上报
```

模型只是链路中的一段。采样频率抖动、丢样、单位换算错误都会让模型效果变差。

## 性能优化

- 使用 int8 量化。
- 降低输入尺寸。
- 减少模型层数和通道数。
- 使用 CMSIS-NN 等优化内核。
- 批量处理传感器窗口。
- 只在需要时唤醒推理。
- 对结果做滞回和置信度阈值。

## OTA 与模型更新

模型更新要考虑：

- 模型版本。
- 输入特征版本。
- 算子兼容。
- 签名校验。
- 回滚。
- A/B 存储。
- 模型和固件是否独立升级。

如果模型输入预处理改变，仅升级模型可能会导致推理结果异常。

## 风险

| 风险 | 对策 |
|---|---|
| 实验数据准确率高，现场差 | 采集现场数据，做持续评估 |
| RAM 放不下 | 量化、裁剪、减小窗口、换平台 |
| 推理太慢 | 优化算子、降低频率、使用硬件加速 |
| 误报/漏报 | 阈值、滞回、结果平滑、人工复核 |
| 难以解释 | 保存输入摘要和推理置信度 |

## 练习

1. 选一个三轴加速度计场景，画出从采样到推理的完整链路。
2. 估算一个 20 KB 模型和 60 KB tensor arena 对 MCU RAM/Flash 的影响。
3. 设计模型 OTA 元数据，包含模型版本、输入版本和签名。

## 工程洞察

TinyML 的难点不是把模型转换成 C 数组，而是让数据、模型、固件和产品语义保持一致。模型在训练集上准确，不代表在目标传感器、目标采样率、目标安装方式、目标温度和真实用户行为下可靠。边缘 AI 把传统嵌入式的资源约束和机器学习的数据分布问题叠加在一起。

设计 TinyML 系统时要明确：

- 输入契约：采样频率、量纲、滤波、归一化、窗口长度和缺失值处理。
- 模型契约：版本、输入 shape、量化方式、tensor arena、延迟、置信度和阈值。
- 输出契约：推理结果如何变成业务动作，误报和漏报代价分别是什么。
- 更新契约：模型 OTA 是否和固件独立，输入预处理变化是否要求模型版本同步。
- 诊断契约：现场如何保存输入摘要、置信度、模型版本和触发上下文。

开放问题：

1. 如果模型准确率提高 2%，但推理耗时翻倍、RAM 增加 40 KB，产品是否值得接受？
2. 传感器更换供应商后，模型是否仍有效？需要哪些回归数据证明？
3. 如果模型输出会触发执行器动作，是否需要规则保护、人工确认或独立传感器交叉验证？
