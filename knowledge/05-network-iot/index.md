# 网络与 IoT 系统

联网设备的核心难题不是“连接成功”，而是在链路不稳定、消息重复、时间不可信、云端不可用和设备长期无人值守时，仍然维护本地安全、数据语义和可诊断性。

本模块建立在 MCU/RTOS 或 Embedded Linux 系统基础之上。网络协议只能传递字节和消息，不能替你定义业务成功、物理动作完成、配置一致性或升级安全。

## 模块契约

### 进入条件

- 已完成至少一条系统主线的基础阶段。
- 能实现有超时的本地通信，并理解状态机、buffer 和持久化基础。
- 能区分字节流、数据报、消息、状态、事件和命令。

### 完成后应能

- 根据距离、吞吐、实时性、功耗、部署和运维选择链路与协议。
- 正确处理 TCP/UART 字节流的分片、合并、短读短写和有界帧长。
- 设计网络、认证、订阅和业务在线的分层连接状态机。
- 为弱网、DNS/TLS 失败、重连、退避和离线缓存定义策略。
- 区分传输 ACK、协议 ACK、业务处理和物理反馈。
- 对命令建立 request ID、幂等/去重、授权、过期和结果查询语义。
- 设计设备身份、密钥、时间、证书、调试口和退役生命周期。
- 设计可验证、可回滚、可诊断的 OTA 与配置迁移。

### 本模块不解决

- 不用 MQTT QoS 或 TCP 可靠性替代端到端业务语义。
- 不用 TLS 连接成功替代本地授权、安全启动和执行器保护。
- 不承诺某一无线制式普遍最优；覆盖、法规、运营和功耗需按产品验证。

## 数据语义先于协议选择

| 数据类型 | 典型例子 | 丢失 | 重复 | 过期 | 缓存满策略 |
|---|---|---|---|---|---|
| 状态 | 当前温度、开关状态 | 可由新值覆盖 | 通常可接受 | 旧值应标记 | 保留最新 |
| 事件 | 告警、故障、开门记录 | 可能不可接受 | 需要事件 ID 去重 | 仍可能有审计价值 | 持久化/分级丢弃 |
| 命令 | 开阀、复位、参数设置 | 需要明确结果 | 非幂等动作危险 | 必须有期限 | 拒绝并告警 |
| 文件 | 固件、模型、证书 | 分片可重传 | 按块校验 | 版本决定 | 暂停或清理 |
| 诊断 | 日志、dump、指标 | 允许分级采样 | 可接受 | 依问题窗口而定 | 限速和优先级 |

协议选型从这张表开始，而不是从“大家都用 MQTT”开始。

## 贯穿项目：可离线工作的远程设备

项目至少包含：

- 本地传感器和一个安全相关执行器或模拟执行器。
- 网络、TLS、会话和业务在线的分层状态。
- 遥测、告警、命令和诊断四类不同语义。
- 断网缓存、退避、时间同步和证书状态。
- 远程配置、命令结果查询和 OTA 预留。

### 必须推演的失败

1. 设备收到命令并执行，但 ACK 前断电。
2. retained 或离线命令在数小时后到达。
3. 云端认为设备关闭，但继电器物理粘连。
4. DNS 正常但 TLS 因时间或证书失败。
5. 网络反复抖动导致重连风暴和电池快速消耗。
6. 离线时间从假设的 10 分钟扩大到 48 小时。
7. OTA 下载完成，但新固件无法恢复业务连接。

每个失败都要区分触发、机制、可见症状、检测证据和恢复动作。

## 必做实验与证据

- 运行 `labs/byte-stream-parser`，验证每个切分点、合并帧和畸形长度。
- 用本地 broker 或测试服务记录连接、订阅、发布和命令状态。
- 注入 DNS 失败、broker 停止、网络抖动和高延迟，检查退避与功耗。
- 重发同一 request ID，证明非幂等动作不会重复执行。
- 让缓存达到容量上限，验证状态、事件和命令采用不同策略。
- 抓包并关联设备状态机日志，明确两个时间源的对齐方式。
- 对证书过期、设备时间错误和服务器身份不匹配建立不同错误状态。

## 定量预算

至少完成三项：

```text
离线缓存 = 生产速率 x 最大离线时间 x 每条大小 + 元数据
重连能量 = 每次连接平均能量 x 失败频率
上报带宽 = payload + 协议/TLS 开销 + 重传
去重容量 = 命令速率 x 去重窗口 x 每条记录大小
证书与 TLS RAM = 握手峰值 + 会话 + 应用 buffer
```

当计算结果超出 RAM、Flash、带宽或电池预算时，不要只调参数；重新设计数据分级、持久化、连接周期或产品承诺。

## 阶段门 G4-G5

- [ ] 字节流 parser 对分片、合并、超长、截断和 EOF 有明确行为。
- [ ] 链路、协议、业务和物理动作成功被分别定义和观测。
- [ ] 状态、事件、命令、文件和诊断具有不同缓存与失败策略。
- [ ] 重连有抖动退避、上限、复位条件和可观测指标。
- [ ] 命令具有身份、授权、过期、request ID、幂等/去重和结果语义。
- [ ] 设备断网或云端不可用时仍维护本地安全状态。
- [ ] TLS、设备身份、密钥存储、安全启动和 OTA 签名边界没有混淆。
- [ ] OTA 任意阶段失败后至少保留一个可信可启动镜像和可诊断结果。

## 迁移与开放问题

1. 从 Wi-Fi 换成蜂窝网络后，连接时延、流量成本、NAT、功耗和离线假设如何改变？
2. 从 MQTT 换成 HTTP 并不会消除哪些业务幂等和命令结果问题？
3. 设备规模从 10 台变成 10 万台时，重连退避、证书轮换和 OTA 灰度什么先成为主导约束？
4. 删除执行器反馈传感器以降低成本后，软件还能证明什么，必须公开接受什么残余风险？

本模块的出口不是“能上云”，而是在不可靠外部环境中仍能解释和控制设备行为。
