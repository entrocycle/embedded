# 网络与 IoT

联网设备的难点不只是“能连上网”，而是弱网、断电、重连、安全、升级、数据一致性和长期运行。

## 协议分层

```text
应用层：MQTT / HTTP / CoAP / 自定义协议
传输层：TCP / UDP / TLS / DTLS
网络层：IPv4 / IPv6 / ICMP
链路层：Ethernet / Wi-Fi / BLE / LoRa / NB-IoT / CAN
物理层：电平、射频、差分信号、线缆
```

## 设计重点

- 设备身份：序列号、证书、密钥、激活。
- 连接管理：配网、重连、心跳、退避。
- 数据上报：周期、事件、缓存、去重。
- 命令下发：确认、超时、幂等。
- OTA：签名、分片、断点续传、回滚。
- 安全：TLS、密钥保护、防重放、防降级。

联网能力上线前，至少要完成设备身份、密钥、TLS、OTA 签名和生命周期管理设计。
真实设备还必须设计 [连接状态机与弱网恢复](connectivity-state-machine.md)，避免把“连上一次”误认为“联网能力可上线”。

## 深挖问题

- 设备离线时，哪些数据必须保留，哪些数据可以丢弃，依据是什么？
- MQTT QoS、应用 ack、幂等 request_id 分别解决什么问题，不能解决什么问题？
- TLS 保护传输，但设备身份、密钥存储和调试口策略还需要怎么设计？
- OTA 成功的定义是什么：下载成功、校验成功、首次启动成功，还是业务恢复成功？
