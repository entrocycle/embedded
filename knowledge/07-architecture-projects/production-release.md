# 量产、测试与发布

从 Demo 到产品，中间差的是可重复制造、可测试、可追溯、可升级、可诊断。量产问题不提前设计，后期会用大量人工补洞。

## 版本管理

至少区分：

- 硬件版本：PCB 版本、BOM 版本。
- Bootloader 版本。
- App 固件版本。
- 协议版本。
- 配置版本。
- 产测程序版本。

固件版本建议包含：

```text
MAJOR.MINOR.PATCH+BUILD
git_commit
build_time
target_board
dirty_flag
```

启动日志示例：

```text
bootloader=1.2.0 app=2.4.3+1024 board=hw_b git=abc1234 reset=watchdog
```

## 产测流程

```text
上电
  ↓
下载产测固件
  ↓
写入 SN/密钥/校准参数
  ↓
电源/时钟/存储自检
  ↓
外设测试
  ↓
通信测试
  ↓
功耗测试
  ↓
下载正式固件
  ↓
最终校验
  ↓
上传测试记录
```

## 产测项目

| 项目 | 目的 |
|---|---|
| 电源电压 | 发现焊接、电源芯片、短路问题 |
| 晶振/RTC | 确认时钟可靠 |
| Flash/RAM | 确认存储可读写 |
| GPIO | 确认关键控制脚 |
| UART/I2C/SPI | 确认总线和外设 |
| 传感器 | 确认读数范围 |
| 无线 | 确认 MAC、RSSI、联网 |
| 功耗 | 发现漏电和模组未睡眠 |
| 按键/LED/显示 | 确认交互部件 |

## SN 与校准

SN 设计要求：

- 唯一。
- 可追溯批次。
- 可被设备和云端读取。
- 不依赖外包装。

校准数据要求：

- 带版本。
- 带 CRC。
- 有默认值和失效处理。
- 记录校准时间和工位。

## 配置分区

配置分区建议双副本：

```text
config_slot_a
  version
  sequence
  length
  payload
  crc

config_slot_b
  ...
```

启动时选择：

1. CRC 正确。
2. sequence 最新。
3. version 可迁移。

写入时先写新副本，再切换 sequence，降低断电损坏风险。

## 发布前测试

功能测试：

- 正常采集/控制/通信。
- 配置修改。
- 异常输入。

稳定性测试：

- 72/168 小时长稳。
- 高低温。
- 电源插拔。
- 弱网。
- 外设断开。

升级测试：

- 正常升级。
- 下载中断电。
- 写入中断电。
- 新固件失败回滚。
- 降级策略。

安全测试：

- 非法固件。
- 过期版本。
- 错误证书。
- 未授权命令。

## 发布包内容

一次正式发布应包含：

- 固件 `.bin/.hex`。
- ELF 和 map 文件归档。
- 符号文件。
- 版本说明。
- Flash 分区说明。
- 配置默认值。
- 已知问题。
- 升级路径。
- 回滚路径。
- 测试报告。

## 现场诊断

现场设备至少要能提供：

- 设备 SN。
- 固件版本。
- 复位原因。
- 最近错误码。
- 网络状态。
- 升级状态。
- 关键计数器。
- 最近一次异常时间。

可通过：

- 串口命令。
- BLE 诊断。
- 云端日志。
- 本地故障文件。
