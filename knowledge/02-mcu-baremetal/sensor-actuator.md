# 传感器与执行器

真实设备不是只有 MCU。嵌入式软件需要把物理世界转成数据，再把控制决策转成电机、继电器、灯光、阀门、蜂鸣器等动作。传感器和执行器是从“会写外设驱动”走向“会做设备”的关键。

## 传感器链路

典型采集链路：

```text
物理量
  -> 传感器
  -> 模拟/数字接口
  -> 驱动
  -> 校准/滤波
  -> 状态机/算法
  -> 上报/控制
```

每个环节都可能引入误差：

- 传感器精度和温漂。
- 供电噪声。
- ADC 参考电压。
- I2C/SPI 通信错误。
- 采样周期不稳定。
- 滤波算法延迟。
- 校准参数丢失。

## 传感器分类

| 类型 | 示例 | 关注点 |
|---|---|---|
| 环境 | 温湿度、气压、光照、气体 | 响应时间、校准、长期漂移 |
| 运动 | 加速度计、陀螺仪、磁力计 | 采样率、坐标系、融合算法 |
| 电量 | 电压、电流、功率计 | 分流电阻、量程、隔离 |
| 距离 | 超声波、ToF、红外 | 盲区、反射面、串扰 |
| 位置 | 编码器、霍尔、GPS | 精度、丢脉冲、时间同步 |

## 传感器驱动要素

一个工程化传感器驱动至少包含：

- 芯片 ID 检查。
- 上电延时。
- 配置寄存器。
- 数据读取。
- 单位换算。
- 超时和总线错误处理。
- 自检或合理范围检查。
- 校准参数接口。
- 低功耗进入和恢复。

接口示例：

```c
typedef struct {
    float temperature_c;
    float humidity_rh;
    uint32_t timestamp_ms;
} env_sample_t;

typedef enum {
    SENSOR_STATUS_OK = 0,
    SENSOR_STATUS_NOT_FOUND,
    SENSOR_STATUS_BUS_ERROR,
    SENSOR_STATUS_TIMEOUT,
    SENSOR_STATUS_DATA_INVALID,
} sensor_status_t;

sensor_status_t env_sensor_init(void);
sensor_status_t env_sensor_read(env_sample_t *sample);
sensor_status_t env_sensor_sleep(void);
sensor_status_t env_sensor_wakeup(void);
```

## 采样与滤波

常见滤波方式：

| 方法 | 特点 | 适用 |
|---|---|---|
| 均值滤波 | 简单，能降低随机噪声 | 慢变量 |
| 中值滤波 | 抑制尖峰 | 按键、距离、ADC 毛刺 |
| 滑动窗口 | 平滑连续数据 | 温度、电压 |
| 一阶低通 | 计算量低 | 控制输入 |
| 卡尔曼/互补滤波 | 融合多传感器 | 姿态估计 |

滤波不是越强越好。滤波会增加延迟，控制系统里可能导致响应变慢或振荡。

## 校准

校准要明确：

- 校准对象：偏置、比例、非线性、温漂。
- 校准时机：出厂、开机、运行中。
- 存储位置：Flash、EEPROM、OTP、文件系统。
- 数据格式：版本、CRC、默认值。
- 失效策略：校验失败时是否使用默认值。

示例：

```text
raw_adc -> voltage -> calibrated_value
calibrated = raw * scale + offset
```

## 执行器链路

典型控制链路：

```text
目标值
  -> 控制算法
  -> 限幅/保护
  -> PWM/GPIO/CAN/RS-485
  -> 驱动电路
  -> 电机/阀门/继电器/灯
  -> 反馈传感器
```

执行器比传感器更需要保护，因为它可能带来热、机械、电流和安全风险。

## 常见执行器

| 执行器 | 控制方式 | 风险 |
|---|---|---|
| LED/背光 | GPIO/PWM/恒流驱动 | 过流、热 |
| 继电器 | GPIO + 三极管/MOS | 反灌、触点寿命 |
| 蜂鸣器 | GPIO/PWM | 驱动电流 |
| 直流电机 | PWM + H 桥 | 堵转、反电动势 |
| 步进电机 | 脉冲 + 方向 | 丢步、发热 |
| 舵机 | 50 Hz PWM | 供电峰值 |
| 电磁阀 | MOS/驱动芯片 | 反向电压、温升 |

## 保护策略

- 输出限幅。
- 过流、过压、过温保护。
- 看门狗和失控关闭。
- 上电默认安全态。
- 故障时进入 fail-safe。
- 执行动作加超时。
- 对关键输出做反馈确认。

## 控制任务设计

执行器控制任务通常要分成：

- 命令解析。
- 安全检查。
- 控制算法。
- 输出驱动。
- 反馈采集。
- 故障处理。

不要让网络命令直接写 PWM 或 GPIO。中间必须经过状态机和安全边界。

## 调试方法

传感器：

- 读芯片 ID。
- 抓 I2C/SPI 波形。
- 固定输入条件，看输出是否稳定。
- 用参考仪表比对。
- 断开传感器验证错误路径。

执行器：

- 空载先测 PWM/GPIO。
- 再接驱动电路。
- 最后接真实负载。
- 测峰值电流和温升。
- 故意制造异常，验证保护。
