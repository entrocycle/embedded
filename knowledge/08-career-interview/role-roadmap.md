# 岗位路线与能力准备

嵌入式岗位名称相近，但能力侧重点差异很大。准备面试前，先明确目标岗位，再组织项目和知识点。

## 常见岗位

| 岗位 | 核心能力 | 项目关键词 |
|---|---|---|
| MCU 软件工程师 | C、外设、裸机、低功耗、量产 | STM32、传感器、Bootloader、产测 |
| RTOS 工程师 | 多任务、实时性、同步、调试 | FreeRTOS、RT-Thread、队列、看门狗 |
| Linux BSP 工程师 | U-Boot、Kernel、DTS、驱动 | bring-up、platform、I2C、RootFS |
| Linux 应用工程师 | 用户态服务、网络、系统集成 | systemd、MQTT、日志、OTA |
| IoT 工程师 | 云接入、弱网、安全、生命周期 | MQTT、TLS、证书、断线重连 |
| 驱动工程师 | 数据手册、总线、内核模型 | 字符设备、input、iio、v4l2 |
| 边缘 AI 工程师 | 模型部署、性能、数据链路 | TinyML、TFLM、量化、传感器 |
| 测试/产测工程师 | 自动化、夹具、质量追溯 | HIL、产测、日志、故障分析 |

## MCU/RTOS 面试准备

必须讲清：

- 复位到 `main()` 的过程。
- `.data`、`.bss`、栈、堆、Flash/RAM 布局。
- `volatile`、中断共享变量和临界区。
- UART/I2C/SPI/ADC/DMA 的调试方法。
- FreeRTOS 任务、队列、信号量、互斥锁和优先级反转。
- 看门狗为什么不能随便喂。
- 低功耗进入、唤醒和功耗测量。
- Bootloader 跳转和 OTA 失败场景。

项目建议：

- 环境监测节点。
- BLE/LoRa 低功耗设备。
- 电机或执行器控制。
- Bootloader + OTA。

## Linux/BSP 面试准备

必须讲清：

- BootROM -> SPL -> U-Boot -> Kernel -> RootFS。
- U-Boot 环境变量和 bootargs。
- 设备树节点如何和驱动匹配。
- platform、I2C、SPI 驱动 probe 流程。
- 字符设备和用户态访问。
- RootFS 构建和 init/systemd。
- `dmesg`、`/proc`、`/sys`、`strace`、`gdbserver` 调试。

项目建议：

- 启动一块 Linux 开发板并修改 DTS。
- 写一个简单 I2C 传感器驱动。
- Buildroot 构建 RootFS。
- 用户态服务采集并上报数据。

## IoT/产品化面试准备

必须讲清：

- 设备身份和激活。
- MQTT topic、QoS、keepalive、Last Will。
- TLS 和证书基本流程。
- 断线重连和退避。
- 离线缓存和队列上限。
- OTA 签名、分片、断电保护和回滚。
- 远程日志和现场诊断。

项目建议：

- MQTT 设备状态机。
- 云端命令下发和执行确认。
- OTA 失败矩阵。
- 弱网长稳测试。

## 项目深挖清单

每个项目都准备这些问题：

- 为什么这样分层？
- 最难的问题是什么？
- 如何复现？
- 你看了哪些日志、波形或寄存器？
- 根因是什么？
- 修复后如何验证？
- 如果量产后复发，如何远程诊断？
- 有没有更好的方案？

## 简历表达

弱表达：

```text
熟悉 STM32、FreeRTOS、MQTT。
```

强表达：

```text
基于 STM32 + FreeRTOS 设计环境监测节点，负责 I2C 传感器驱动、采集任务、MQTT 上报和看门狗健康监控。实现断线重连、离线缓存上限和 OTA 结果上报，完成 72 小时弱网长稳测试。
```

强表达包含：

- 平台。
- 职责。
- 关键技术。
- 工程约束。
- 可验证结果。

3. 准备一个“最难问题”的 3 分钟讲述版本。

## 成长洞察

岗位路线不是给自己贴标签，而是帮助你选择训练重点。MCU/RTOS、Linux/BSP、IoT/产品化之间有不同主线，但底层能力相通：读文档、建模型、做实验、收集证据、管理资源、处理失败、写可维护代码。

规划成长时避免两个极端：

- 只追新技术：会很多框架，但无法解释启动、内存、时序和故障机制。
- 只钻单点细节：能优化某个寄存器配置，但缺少架构、验证和产品化视角。

更稳的路径是“T 型”：选择一个方向做深，同时保持跨层理解。比如 MCU 工程师至少要理解一点 Linux/IoT 的系统边界，Linux 驱动工程师也要理解硬件时序和电源，IoT 工程师不能只懂云协议而不懂本地安全态和 OTA 恢复。

每隔一段时间，用项目产出评估自己：我是否能独立完成从需求澄清、方案取舍、实现、调试、验证到复盘的闭环？能闭环的问题越复杂，岗位能力越扎实。
