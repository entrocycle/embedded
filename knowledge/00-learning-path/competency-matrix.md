# 嵌入式能力矩阵

学习嵌入式最容易出现的问题，是把知识点看成孤立条目：会一点 C、会点灯、会调一个 HAL API、会建一个 FreeRTOS 任务。能力矩阵的作用，是把这些条目放回真实工程岗位里，明确每个阶段应该能独立产出什么。

## 能力分层

| 层级 | 关注对象 | 初级要求 | 进阶要求 | 高级要求 |
|---|---|---|---|---|
| 语言与内存 | C/C++、栈、堆、链接段 | 会写安全的 C 函数 | 能解释内存布局和 map 文件 | 能设计受限资源下的内存策略 |
| 硬件认知 | 原理图、数据手册、电气特性 | 能看懂最小系统 | 能定位接口和电源问题 | 能参与硬件方案评审 |
| 裸机驱动 | GPIO、UART、SPI、I2C、ADC、DMA | 能跑通外设 | 能封装驱动和错误处理 | 能做跨平台驱动抽象 |
| 实时系统 | 中断、任务、调度、同步 | 能使用 RTOS API | 能分析优先级和栈问题 | 能设计实时任务模型 |
| Linux/BSP | U-Boot、Kernel、DTS、RootFS | 能启动板卡和改 DTS | 能写基础驱动和系统服务 | 能维护产品级 BSP |
| 联网与 IoT | TCP/IP、MQTT、BLE、OTA、安全 | 能接入网络 | 能处理弱网和升级失败 | 能设计设备生命周期 |
| 调试优化 | 日志、GDB、仪器、功耗 | 能复现并定位普通问题 | 能跨层收集证据 | 能建立诊断体系和回归测试 |
| 工程交付 | 架构、构建、测试、发布 | 能按规范提交代码 | 能设计模块和 CI | 能主导量产、现场维护和复盘 |

## 三条职业路线

### MCU/RTOS 工程师

核心能力：

- 熟悉 Cortex-M 或同类 MCU 的启动、中断、时钟和外设。
- 能用 C 写可靠的驱动、状态机和通信协议。
- 能使用 FreeRTOS、RT-Thread 等 RTOS 搭建多任务系统。
- 能处理低功耗、看门狗、Bootloader、OTA 和产测问题。

代表产出：

- 一个最小 BSP。
- 一组平台无关传感器驱动。
- 一个带日志、配置、OTA、看门狗的 RTOS 应用。
- 一份故障复盘和功耗报告。

### 嵌入式 Linux/BSP 工程师

核心能力：

- 理解 BootROM、SPL、U-Boot、Kernel、DTB、RootFS 的启动链路。
- 能修改设备树、内核配置、驱动和用户空间服务。
- 能使用 Buildroot/Yocto 构建系统。
- 能定位启动失败、驱动 probe 失败、文件系统挂载失败和性能问题。

代表产出：

- 一个可启动 Linux 镜像。
- 一个 platform/I2C/SPI 驱动。
- 一套 rootfs、systemd 服务和日志采集方案。
- 一份 BSP bring-up 记录。

### IoT/边缘设备工程师

核心能力：

- 理解设备身份、配网、云接入、协议、断线重连和 OTA。
- 能设计弱网下可恢复的数据上报和命令下发。
- 能处理 TLS、证书、密钥、签名和回滚。
- 能把硬件、固件、云平台和运维诊断串成闭环。

代表产出：

- MQTT/HTTP/CoAP 接入状态机。
- 设备激活和密钥管理流程。
- OTA 分片、校验、回滚方案。
- 现场诊断和远程日志方案。

## 阶段验收表

| 阶段 | 建议周期 | 关键实验 | 必须输出 |
|---|---:|---|---|
| 入门 | 2-4 周 | 点灯、串口日志、map 文件分析 | 内存布局笔记、工程构建记录 |
| 裸机 | 4-8 周 | UART 环形缓冲、I2C 传感器、ADC + DMA | 外设驱动接口、波形截图、错误码 |
| RTOS | 4-8 周 | 多任务采集、队列通信、看门狗心跳 | 任务表、栈统计、故障注入报告 |
| Linux | 6-12 周 | U-Boot 启动、DTS 修改、字符驱动 | 启动日志、RootFS 构建脚本、驱动说明 |
| IoT | 4-8 周 | MQTT 断线重连、TLS、OTA | 连接状态机、升级失败矩阵 |
| 工程化 | 持续 | CI、长稳、产测、现场诊断 | 发布清单、测试报告、复盘文档 |

## 自查问题

每学完一个专题，都用下面的问题检查是否真正掌握：

- 这个模块解决什么问题，不解决什么问题？
- 它依赖哪些硬件资源、时钟、中断、内存和电源状态？
- 它失败时有哪些可观测信号？
- 它和上下层通过什么接口交互？
- 它在量产和现场维护时如何被测试、升级和诊断？

## 能力缺口定位

如果遇到问题，可以用下面的症状反推学习缺口：

| 症状 | 常见缺口 |
|---|---|
| 只会照着例程改，不敢删 HAL 代码 | 启动、时钟、寄存器和数据手册方法不足 |
| 外设偶发失败但不知道看哪里 | 波形、错误标志、超时和状态机不足 |
| RTOS 项目经常随机死机 | 栈、优先级、锁、队列和 ISR 规则不足 |
| Linux 能启动但驱动不 probe | 设备树、总线匹配和内核日志不足 |
| OTA 跑通一次但不敢上线 | 断电保护、签名、回滚和版本兼容不足 |
| 面试项目讲不深 | 缺少架构图、定位过程和工程复盘 |
