# 学习路线总览

嵌入式软件不是单一技能，而是一组跨硬件、系统、软件工程和产品约束的能力组合。学习时最容易走偏的地方，是只学某块开发板或某个库，却没有建立“系统如何从上电到运行”的主线。

## 能力模型

| 层级 | 核心问题 | 必备能力 | 产出物 |
|---|---|---|---|
| C 与基础 | 程序如何变成机器可执行的固件 | C、指针、内存、编译链接、Make/CMake | 可解释 map 文件和内存布局 |
| MCU 裸机 | CPU 如何启动，外设如何被控制 | 启动流程、寄存器、时钟、中断、GPIO/UART/SPI/I2C/ADC/DMA | 裸机驱动和最小 BSP |
| RTOS | 多任务如何确定性运行 | 任务、调度、队列、信号量、互斥、定时器、ISR 协作 | FreeRTOS/RT-Thread 应用 |
| Linux | 复杂 SoC 如何运行完整系统 | U-Boot、Kernel、DTS、RootFS、字符设备、总线驱动 | 可启动 Linux 镜像和基础驱动 |
| 网络 IoT | 设备如何可靠连接外部世界 | TCP/IP、MQTT、HTTP、BLE、LoRa、CAN、OTA、安全 | 可联网设备和升级链路 |
| 工程化 | 如何稳定交付产品 | 调试、测试、日志、CI、架构、功耗、异常恢复 | 可维护项目模板 |

更细的岗位能力拆解见 [嵌入式能力矩阵](competency-matrix.md)。

学习每个主题时，建议先阅读 [如何把本教程学深](deep-learning-method.md)。它会把“看章节”升级为“做实验、找证据、做取舍、写复盘”的工程训练流程。

## 四阶段路线

### 阶段一：打底

目标是能读懂一个嵌入式 C 工程从源码到固件的完整过程。

- C 语言：指针、数组、结构体、位操作、`const`、`volatile`、`static`、内存对齐。
- 编译链接：预处理、编译、汇编、链接、链接脚本、启动文件、ELF/HEX/BIN。
- 计算机基础：栈、堆、中断、内存映射 I/O、总线、时钟、复位。

验收标准：

- 能说明一个全局变量、局部变量、`static` 变量分别放在哪里。
- 能读懂 `.map` 文件中 Flash/RAM 占用。
- 能解释为什么寄存器指针通常要加 `volatile`。

### 阶段二：掌握 MCU 裸机

目标是脱离图形化配置工具，也能理解外设初始化和数据流。

- Cortex-M 启动流程：向量表、复位处理函数、栈指针、`.data` 拷贝、`.bss` 清零。
- 外设驱动：GPIO、UART、SPI、I2C、ADC、PWM、DMA、RTC、Watchdog、CAN。
- 中断：NVIC、优先级、临界区、ISR 与主循环协作。

验收标准：

- 能从数据手册定位外设基地址、寄存器偏移和位定义。
- 能写一个 UART 环形缓冲接收驱动。
- 能用逻辑分析仪验证 SPI/I2C/UART 时序。

### 阶段三：进入系统开发

目标是掌握 RTOS 和 Linux 两条路线的基本工程能力。

- RTOS：任务模型、优先级、队列、信号量、互斥锁、事件组、软件定时器、内存管理。
- Linux：U-Boot、设备树、内核裁剪、驱动模型、RootFS、系统服务。
- 系统边界：实时性、内存、功耗、启动时间、文件系统可靠性。

验收标准：

- 能判断一个功能应该放在中断、任务还是定时器回调中。
- 能解释优先级反转，以及互斥锁为什么需要优先级继承。
- 能为一个 Linux 外设补 DTS 节点并写基础 platform driver。

### 阶段四：工程与产品化

目标是具备交付真实设备的能力。

- 网络：MQTT/HTTP/CoAP、TLS、证书、断线重连、心跳、数据缓存。
- OTA：分区、版本、签名、回滚、断电保护。
- 调试优化：GDB/OpenOCD、串口日志、SWO/RTT、示波器、功耗分析。
- 工程管理：BSP 分层、模块接口、单元测试、CI、产测、日志和故障码。

验收标准：

- 能设计一个设备从出厂、激活、运行、升级到故障恢复的完整链路。
- 能通过日志、波形、寄存器和 dump 信息定位问题。
- 能写出清晰的模块接口和项目复盘。

## 路线选择

### MCU/RTOS 方向

适合智能硬件、工业控制、可穿戴、低功耗设备。重点是实时性、资源受限、外设控制和功耗。

推荐路径：

`C 基础 -> Cortex-M -> 外设驱动 -> FreeRTOS -> 网络协议 -> 低功耗 -> 项目实战`

### 嵌入式 Linux/BSP 方向

适合网关、摄像头、HMI、机器人、车载、工业边缘设备。重点是启动链路、内核、设备树、驱动和系统集成。

推荐路径：

`C/Linux 基础 -> U-Boot -> Kernel/DTS -> 字符设备 -> 总线驱动 -> RootFS -> 系统调试`

### IoT/边缘智能方向

适合联网设备、云端联动、边缘 AI。重点是网络、安全、OTA、数据链路和模型部署。

推荐路径：

`MCU 或 Linux 基础 -> TCP/IP -> MQTT/HTTP -> TLS -> OTA -> TinyML/边缘推理 -> 云平台`

## 每周学习节奏

| 周期 | 输入 | 实践 | 输出 |
|---|---|---|---|
| 周一至周二 | 阅读章节和数据手册 | 复现实验 | 记录关键概念 |
| 周三至周四 | 阅读参考代码 | 修改功能 | 画流程图/时序图 |
| 周五 | 调试和故障注入 | 加日志、测波形 | 写问题清单 |
| 周末 | 做小项目 | 整理仓库 | 写复盘文档 |
