# 调试与优化

嵌入式调试的关键是把问题定位到层次：供电、时钟、复位、下载、启动、外设、协议、调度、内存、网络或业务逻辑。

## 调试工具链

| 工具 | 解决问题 |
|---|---|
| 串口日志 | 启动流程、状态变化、错误码 |
| GDB/OpenOCD | 断点、单步、寄存器、内存 |
| 逻辑分析仪 | UART/SPI/I2C/CAN 时序 |
| 示波器 | 电源、时钟、PWM、模拟信号 |
| 万用表 | 电压、电流、通断 |
| Wireshark | 网络包分析 |
| SystemView/Trace | RTOS 任务和中断行为 |

调试能力要进一步沉淀为 [现场诊断与可观测性](field-diagnostics.md)：版本、复位原因、故障记录、健康指标、远程日志和诊断命令。

## 深挖问题

- 当前问题的根因证据是什么？哪些证据只证明了现象？
- 如果设备已经发到现场，你现在缺少哪些日志、dump、计数器或诊断命令？
- 性能优化前的指标是什么？优化后有没有复测同一基线？
- 功耗异常要按哪些状态拆分，如何区分 MCU、外设、GPIO 和无线模组的贡献？

## 阶段产出

- 一个 HardFault 或异常复位记录，包含 PC/LR/SP/CFSR、版本和复位原因。
- 一份总线波形分析，能对应到代码中的一次读写。
- 一份故障复盘，按现象、假设、证据、根因、修复、预防组织。
