# 典型故障案例库

故障案例的价值在于训练定位思路。下面按现象、证据、根因、处理方式组织。

## 上电无日志

现象：

- 串口无任何输出。
- 调试器偶尔无法连接。

定位顺序：

1. 测 VDD/VDDA 是否正常。
2. 检查 BOOT 引脚。
3. 检查 NRST 是否被拉低。
4. 确认串口 TX/RX 和波特率。
5. 用调试器 halt，看 PC 是否在预期地址。
6. 确认向量表和链接地址。

常见根因：

- Boot 模式进了系统 BootROM。
- 外部晶振未起振，时钟初始化卡死。
- 串口引脚复用错误。
- 固件链接到 `0x08008000`，但未使用 Bootloader 跳转。

## 一进中断就 HardFault

现象：

- 主循环正常。
- 使能某外设中断后崩溃。

定位：

- 查看向量表中对应 IRQ handler 是否存在。
- 查看中断函数名是否和启动文件一致。
- 查看是否清中断标志。
- 查看 ISR 是否访问空指针或未初始化对象。
- 读取 CFSR/HFSR。

常见根因：

- 中断函数名写错，进 Default_Handler。
- 外设中断标志未清，反复进入 ISR。
- ISR 调用了阻塞函数。
- RTOS 中 ISR 调用了非 FromISR API。

## I2C 设备无 ACK

现象：

- 逻辑分析仪看到地址后 NACK。

定位：

1. 确认设备供电。
2. 确认上拉电阻。
3. 确认 7 位地址还是 8 位地址。
4. 确认 SCL/SDA 是否接反。
5. 确认设备是否需要延迟启动。
6. 扫描 I2C 地址。

常见根因：

- 把 7 位地址左移后又交给 HAL，导致地址错误。
- 传感器地址脚电平与代码不一致。
- I2C 总线无上拉。
- 设备还未完成上电复位。

## SPI 读到全 0xFF 或 0x00

现象：

- 写命令有波形，读数据恒定。

定位：

- CS 是否有效。
- MISO 是否连接。
- CPOL/CPHA 是否正确。
- 读命令后是否需要 dummy byte。
- 芯片是否处于 power-down。

常见根因：

- CPHA 配错。
- CS 提前拉高。
- SPI 模式和器件手册不一致。
- MISO 被其他设备占用。

## UART 丢包

现象：

- 小数据正常，大量数据丢失。
- 日志输出影响接收。

定位：

- 看是否使用阻塞接收。
- 检查 RX buffer 大小。
- 统计 overrun 错误。
- 检查中断优先级。
- 检查处理任务是否来不及消费。

处理：

- 中断或 DMA 接收。
- 环形缓冲。
- 高水位统计。
- 协议层加入帧头、长度、CRC。

## RTOS 随机死机

现象：

- 运行几小时后无响应。
- 复现不稳定。

定位：

- 开启栈溢出检测。
- 记录每个任务 high water mark。
- 记录 heap 最小值。
- 统计队列满。
- trace 任务状态。
- 检查死锁和优先级反转。

常见根因：

- 某任务栈过小。
- 日志队列满后阻塞高优先级任务。
- 低优先级任务持锁时间过长。
- 动态内存碎片。

## OTA 后无法启动

现象：

- 下载完成重启后 Bootloader 不跳转。
- 或跳转后 HardFault。

定位：

- 检查固件头 magic/size/hash。
- 检查 App 链接地址。
- 检查向量表第 0 项是否在 RAM。
- 检查 Reset_Handler 是否在 App Flash 范围。
- 检查 VTOR/MSP 设置。

常见根因：

- App 仍链接到 `0x08000000`。
- 下载区和运行区混淆。
- 固件写入偏移错误。
- Bootloader 未关闭中断。

## 低功耗电流偏高

现象：

- Stop 模式电流远高于预期。

定位：

- 测整机和各模块分支电流。
- 逐个关闭外设和 GPIO。
- 断开外部模块验证。
- 检查调试器是否连接。
- 检查是否进入了真正的低功耗模式。

常见根因：

- GPIO 悬空。
- 外设时钟未关闭。
- 传感器/无线模组未 sleep。
- IO 反向供电。
- RTOS tick 周期唤醒。
