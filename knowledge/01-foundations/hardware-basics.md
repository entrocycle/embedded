# 硬件基础与数据手册阅读

嵌入式软件工程师不一定要会完整画板，但必须能看懂原理图、数据手册和关键电气约束。很多软件问题的根因实际来自供电、时钟、复位、上拉、电平和接口时序。

## 本章任务与边界

完成本章后，你应能检查电压域、峰值电流、复位/启动脚、GPIO 电气属性、I2C 上拉和 ADC 输入条件，并为“无法启动、随机复位、通信不稳、采样异常”选择合适的电气证据。

本章面向软件工程师建立协作能力，不授权在缺乏安全知识时测量市电、高压、大电流或危险执行器。涉及探头接地、量程、反向电动势和电池操作时，应遵循仪器、板卡和实验室安全要求。

完成证据：板卡电源树、关键引脚默认状态表，以及一次带测量条件的电源/信号实验；仅记录“万用表显示 3.3 V”不足以证明瞬态供电可靠。

## 必懂硬件概念

| 概念 | 软件工程师需要知道什么 |
|---|---|
| 电压域 | MCU、传感器、模组可能工作在不同电压，GPIO 不能随意直连 |
| 电流能力 | GPIO 只能驱动小电流，电机、继电器、背光需要驱动电路 |
| 上拉/下拉 | I2C、按键、启动脚、复位脚常依赖默认电平 |
| 去耦电容 | 抑制电源瞬态噪声，布局不当会导致随机复位 |
| 晶振 | 决定时钟稳定性，影响 USB、通信波特率、RTC |
| 复位 | 上电时序不满足会导致启动失败 |
| 电平转换 | 3.3V 与 5V、1.8V 接口互联需要确认兼容性 |
| ESD/浪涌 | 外部接口需要保护，否则现场环境中容易损坏 |

## 原理图阅读顺序

阅读一块板子的原理图，建议按这个顺序：

1. 电源树：输入电压、LDO/DC-DC、每路输出、电流能力。
2. MCU 最小系统：VDD/VDDA、晶振、复位、BOOT、SWD/JTAG。
3. 时钟：外部晶振频率、RTC 晶振、是否有 MCO。
4. 下载调试：SWDIO/SWCLK/NRST/UART 是否引出。
5. 外设连接：I2C/SPI/UART/CAN/USB/ADC/PWM。
6. 关键 GPIO：LED、按键、电源使能、模组复位、唤醒脚。
7. 存储器：SPI Flash、EEPROM、SD 卡、eMMC。
8. 无线/网络：天线、模组供电、串口/SDIO/SPI 接口。

## 数据手册阅读方法

芯片资料通常包括：

| 文档 | 重点 |
|---|---|
| Datasheet | 引脚、电气特性、封装、内存和外设概览 |
| Reference Manual | 外设寄存器和系统架构 |
| Programming Manual | CPU 内核、指令、异常模型 |
| Errata | 芯片已知缺陷和规避方法 |
| Application Note | 官方推荐设计和使用方式 |

阅读新外设时，按这个模板摘录：

```text
外设名称：
挂载总线：
时钟来源：
引脚复用：
寄存器基地址：
初始化顺序：
关键状态位：
中断源：
DMA 请求：
错误标志：
低功耗影响：
已知限制/Errata：
```

## GPIO 电气属性

GPIO 不只是高低电平，还包括：

- 输入阈值：什么电压算高，什么电压算低。
- 输出类型：推挽、开漏。
- 驱动能力：最大灌电流/拉电流。
- 速度等级：影响上升沿和 EMI。
- 上拉/下拉：内部电阻通常很弱，不能替代所有外部上拉。
- 5V tolerant：不是所有引脚都能容忍 5V。

开漏输出适合：

- I2C 总线。
- 多设备线与逻辑。
- 电平转换。

推挽输出适合：

- 普通 LED。
- 片选信号。
- PWM 控制信号。

## I2C 上拉电阻

I2C 依赖上拉电阻，阻值必须同时满足器件拉低能力和总线上升时间，不能只背 `4.7k` 这个经验值。

下界由最弱器件允许的低电平输出电流决定：

```text
Rmin = (VDD - VOL(max)) / IOL
```

上界可用 RC 上升时间近似估算：

```text
Rmax = tr(max) / (0.8473 * Cb)
```

其中 `Cb` 包含引脚、走线、连接器、线缆和测量探头电容。以 3.3 V 总线、器件保证 `VOL <= 0.4 V @ 3 mA`、估算 `Cb = 100 pF` 为例：

```text
Rmin = (3.3 V - 0.4 V) / 3 mA = 967 ohm
```

若所用 I2C 模式规定 `tr(max) = 300 ns`，则：

```text
Rmax = 300 ns / (0.8473 * 100 pF) = 3.54 kohm
```

可从标准阻值中选择落在区间内的值，再用示波器在最差电压、温度、线长和器件组合下测量上升时间与低电平电压。`tr(max)`、`VOL(max)` 和 `IOL` 必须查对应 I2C 规范及所有器件数据手册；1 MHz Fast-mode Plus 还要求总线上每个器件都支持该模式，不能仅靠减小电阻提速。

电阻过大时边沿慢，电阻过小时静态低电平电流、功耗和器件灌电流增加。并联多个板载上拉时要按并联等效值重新计算。

## ADC 输入

ADC 采样不是直接读电压这么简单。

要关注：

- 参考电压稳定性。
- 输入阻抗。
- 采样时间。
- RC 滤波。
- 分辨率与有效位数。
- 校准和温漂。

如果传感器输出阻抗较高，ADC 采样时间太短会导致读数偏差。

## 电源和复位问题特征

| 现象 | 可能原因 |
|---|---|
| 无法稳定下载 | 供电不足、NRST 异常、SWD 线过长 |
| 无线发送时复位 | 峰值电流不足、电源压降 |
| 偶发 HardFault | 电源噪声、时钟不稳、栈/内存问题 |
| RTC 不准 | 低速晶振负载电容不匹配 |
| ADC 跳动 | VDDA 噪声、地线布局、采样时间不合理 |

## 软件如何帮助硬件定位

- 启动时打印复位原因。
- 保留电源异常计数。
- 对关键外设加自检。
- 读取芯片唯一 ID 和 option bytes。
- 对 ADC 做内部参考电压测量。
- 对外设错误标志做统计。

## Bring-up 检查清单

- [ ] 输入电压正确。
- [ ] 3.3V/1.8V 等电源轨稳定。
- [ ] NRST 默认为高。
- [ ] BOOT 引脚电平正确。
- [ ] SWD 可连接。
- [ ] 串口 TX 有输出。
- [ ] 外部晶振起振。
- [ ] I2C/SPI/UART 引脚复用正确。
- [ ] 传感器供电和地址正确。
- [ ] 无线模组峰值电流满足。

## 练习

1. 找一块开发板原理图，画出电源树和 MCU 最小系统。
2. 从数据手册中整理一个 UART 或 I2C 外设的初始化清单。
3. 用万用表测量开发板 3.3V，在无线发送或 LED 亮灭时观察是否波动。

## 工程洞察

硬件基础的关键不是记住元件名称，而是理解软件行为如何被物理条件限制。很多“软件偶发问题”最终来自电源瞬态、复位边界、GPIO 浮空、上拉不当、晶振不稳或执行器反灌电。软件可以通过日志和状态机提高可观测性，但不能违反硬件边界。

分析硬件问题时，建议先问：

- 供电是否满足静态电流、峰值电流和瞬态响应，而不只是空载电压正常？
- 复位和启动脚是否在上电、掉电、低功耗唤醒和调试连接时都有确定状态？
- 通信引脚是否存在电平不兼容、过长走线、上拉过弱或总线争用？
- 模拟输入是否满足采样源阻抗、参考电压、滤波和保护要求？
- 执行器是否有反向电动势、峰值电流、温升和失效安全状态设计？

软件工程师参与硬件评审时，不需要替代硬件工程师计算所有参数，但必须能提出会影响固件设计和现场诊断的问题。越早发现这些边界，后续代码越少依赖“经验延时”和“偶然能跑”。
