# 基础能力

嵌入式基础不是“C 语言语法 + 单片机 API”，而是能解释程序在受限硬件上如何被存储、启动、运行和出错。

## 必须掌握的概念

| 概念 | 嵌入式中的意义 |
|---|---|
| 指针 | 访问内存、寄存器、DMA 缓冲、协议数据 |
| `volatile` | 告诉编译器变量可能被外设、中断或其他上下文改变 |
| `const` | 放入只读区，表达不可变配置 |
| `static` | 控制生命周期和链接可见性 |
| 位操作 | 配置寄存器、协议字段、状态标志 |
| 栈 | 函数调用、局部变量、中断上下文，过小会栈溢出 |
| 堆 | 动态内存，嵌入式中要谨慎使用 |
| 对齐 | 影响访问效率、总线错误、协议结构体布局 |
| 链接脚本 | 决定代码和数据放进 Flash/RAM 的位置 |

## 学习验收

- 能写出一个 `volatile uint32_t *` 访问寄存器的例子。
- 能解释 `.text`、`.rodata`、`.data`、`.bss`、heap、stack。
- 能从 `.map` 文件判断 Flash 和 RAM 占用。
- 能说明为什么中断里不应做复杂阻塞操作。
- 能读懂芯片数据手册中的电源、时钟、引脚复用和外设章节。
- 能按模板整理一个外设的数据手册笔记和 bring-up 记录。

## 深挖问题

- 一段 C 代码中，每个对象分别在哪里、谁会访问、生命周期多长？
- 编译器优化、`volatile`、中断和 DMA 之间的边界是什么？
- 链接脚本为什么会决定系统能否启动，而不只是影响固件大小？
- 数据手册里哪些参数是“能跑”的条件，哪些参数是“可靠运行”的条件？

## 阶段产出

- 一份 `.map` 文件分析，标出 Flash/RAM 占用和主要符号。
- 一份外设数据手册摘录，包含寄存器、时序、电气限制和误读风险。
- 一个最小寄存器访问实验，说明 `volatile`、位操作和读改写风险。
